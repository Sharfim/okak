Всё просто! Как только ты внесешь изменения в файлы через VS Code, тебе нужно будет выполнить «золотую троицу» команд в терминале.
Вот твой пошаговый алгоритм на будущее:
1. Подготовка (в папке проекта)
Сначала посмотри, какие файлы изменились (необязательно, но полезно):
bash

git status

Используйте код с осторожностью.
2. Сохранение изменений
Добавь все измененные файлы в очередь на отправку:
bash

git add .

Используйте код с осторожностью.
3. Создание записи (Коммит)
Обязательно напиши кратко, что именно ты изменил, чтобы сам не запутался в истории:
bash

git commit -m "Добавил метод save в класс User"

Используйте код с осторожностью.
4. Отправка на сервер
Так как ты настроил привязку с токеном, просто пиши:
bash

git push origin main

Используйте код с осторожностью.
Что если возникнет ошибка (как сегодня)?
Если ты поработал на одном компе, запушил код, а потом сел за другой и забыл скачать обновления:

    Сначала попробуй скачать то, что на сервере: git pull origin main.
    Если всё запуталось и ты хочешь просто «затереть» версию на GitHub своей текущей: git push origin main --force.

Лайфхак для VS Code
В VS Code слева есть иконка с тремя точками и линиями (Source Control или Ctrl+Shift+G). Там можно нажимать кнопки «+» (вместо add) и галочку (вместо commit), а потом «Sync Changes» — это то же самое, что команды в терминале, только мышкой.
